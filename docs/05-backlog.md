# Innova Mentor

**Última actualización:** Junio de 2026

## Objetivo del documento

Este documento reúne todas las funcionalidades planificadas para Innova Mentor.

Su propósito es servir como hoja de ruta del proyecto, organizando las tareas por prioridad y permitiendo al equipo conocer el estado actual del desarrollo.

El backlog es un documento vivo y será actualizado continuamente durante el ciclo de vida del proyecto.

---

# Estado del proyecto

Actualmente el proyecto se encuentra en la fase de construcción del MVP (Producto Mínimo Viable).

Durante esta etapa el objetivo principal es implementar las funcionalidades esenciales que permitan validar la propuesta de valor de Innova Mentor.

# Criterios de priorización

Las funcionalidades del backlog serán implementadas considerando los siguientes criterios:

1. Valor para el usuario.
2. Impacto en el MVP.
3. Complejidad técnica.
4. Dependencias con otros módulos.
5. Tiempo disponible del equipo.

---

# Gestión del Backlog

El backlog será revisado de forma periódica durante el desarrollo del proyecto.

Cada nueva funcionalidad deberá:

* Analizarse previamente.
* Documentarse.
* Priorizarse.
* Implementarse.
* Probarse.
* Registrarse en el historial de versiones.

---

# Conclusión

El backlog representa la hoja de ruta de Innova Mentor.

Su objetivo no es únicamente registrar tareas pendientes, sino proporcionar una visión clara de la evolución prevista del proyecto, facilitando la planificación, el trabajo colaborativo y la toma de decisiones durante todo el desarrollo.
